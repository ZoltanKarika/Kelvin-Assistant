# Roadmap and Status Mode

Use this mode when asked to inspect the project roadmap and current status.

Goal:

Understand what is already done, what is missing, and what should happen next.

Rules:

- Do not modify files.
- Read roadmap, planning, changelog, README, docs, and relevant source files.
- Separate facts from assumptions.
- Do not invent completed work.

Key files to read:

- `docs/roadmap.md` — full milestone list and acceptance criteria.
- `docs/n8n-integration.md` — v0.7 architecture and implementation order.
- `docs/ai/v07-guide.md` — completed v0.7 progress tracker.
- `docs/ai/v08-guide.md` — completed v0.8 progress tracker.
- `docs/ai/v09-guide.md` — completed v0.9 progress tracker.
- `docs/ai/v10-guide.md` — current v1.0 planning tracker and next steps.

Output format:

1. Current status
2. Completed items
3. Partially completed items
4. Missing items
5. Risks/blockers
6. Recommended next steps
