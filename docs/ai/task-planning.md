# Task Planning Mode

Use this mode when asked to create a todo list or implementation plan.

Goal:

Turn roadmap/status findings into small, safe, testable tasks.

Rules:

- Do not modify code.
- Break large work into small tasks.
- Each task must have a clear outcome.
- Include dependencies.
- Include suggested branch names.
- Include suggested commit messages.
- Prefer one PR per logical feature or fix.

For v0.7 tasks, always reference `docs/ai/v07-guide.md` for the current
progress and the correct step ordering.

Output format:

1. Todo list
2. Priority order
3. Dependencies
4. Suggested branch name
5. Suggested commit plan
6. Testing plan
